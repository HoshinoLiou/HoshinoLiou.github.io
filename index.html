使用 Hexo 與 Github 打造屬於自己的部落格

  1. Hexo 是甚麼? Hexo官網

  2. Github 是甚麼? Github官網


第一步 - 註冊 Github、設定本地資料夾
  1. 首先要 註冊 Github 帳號，並在自己的頁面 新增一個 Repository
  2. Repository 的名稱為 <使用者帳號>.github.io，好比說我的帳號id是 HoshinoLiou， 名稱就會是 HoshinoLiou.github.io。
  3. 在本機端 創建一個名為 <使用者帳號>.github.io 的資料夾 (使用者帳號一樣填入自己的帳號id)，然後依照 Github 提供的指令進行設定。
第二步 - 安裝 Hexo、以及事前準備
  1. 因為要使用 npm 進行 Hexo 的安裝，所以需要先 安裝 Node.js (至 Node.js官網下載)。安裝完成後，在 cmd 中輸入 node –version 以及 npm –version 確認是否安裝成功。(有出現版本資訊，代表有安裝成功)
  1. 使用 npm 進行 Hexo 的安裝，在 cmd 中輸入 npm install hexo-cli -g 進行安裝。安裝完成後，一樣在 cmd 中輸入 hexo –version 確認是否安裝成功。(有出現版本相關資訊，代表有安裝成功)
第三步 - 建立 Hexo 專案、部署於 Github 上
  1. 在 cmd 中輸入 hexo init blog，這個指令會在本地端產生一個名為 blog 的資料夾，資料夾裡面也會產生 Hexo 基本的設定檔案。我們將裡面的檔案 全部複製到第一個步驟建立的 <使用者帳號>.github.io 資料夾中。
  2. 然後在 <使用者帳號>.github.io 資料夾中用 cmd 執行 npm install，就會看到 Hexo 的相關檔案被建立。
  1. 打開 config.yml 進行設定，在 deploy 的部分進行修改。
  1. 更新完 config.yml 後，需要執行 hexo clean 以及 hexo generate。之後可以透過 hexo server 在本地端進行測試，如果確認沒問題就可以使用 hexo deploy 將結果佈署到 Github 上。(如果在執行 deploy 時，出現 ERROR Deployer not found: git 的錯誤，就在 cmd 中執行 npm install hexo-deployer-git –save)
  1. 打開 Github 頁面點選 Settings ，接著點選下方的 GitHub Pages 即可看到結果。
